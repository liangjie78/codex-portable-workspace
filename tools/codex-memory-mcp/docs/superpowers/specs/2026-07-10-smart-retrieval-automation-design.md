# CodexMemory v0.7.0–v0.9.0：智能检索与自动化设计

## 目标

在不引入网络服务、嵌入模型或数据库的前提下，让本机 CodexMemory 更容易被自然语言找到，并把“任务开始时该看什么、记忆库后续该维护什么”变成稳定的只读辅助能力。

现有 v0.6.0 的 22 条评测都使用了卡片中的原始词汇，Top-1 为 100%。额外的自然表达审计暴露了真实缺口：`网络共享路径来源检查`、`知识库写到一半中断怎么处理` 与 `备份没有带上新工具` 不能稳定命中其对应卡片。因此本阶段不把词面重叠误称为语义检索，而是新增可审阅的本地别名并改进排序证据。

## 不做什么

- 不安装 embedding、向量数据库、云端 API 或常驻后台服务。
- 不自动删除、合并、降级、复核或重写任何真实知识卡。
- 不扩大 Codex 的权限，不修改模型、认证、代理或全局安全策略。
- 不改变现有九个 MCP 工具的名字、参数含义或默认行为。

## 数据模型

卡片新增可选前置字段 `aliases`：非空、去重的字符串数组，最多 12 项，每项最多 120 个字符且不含换行。它表示卡片可被怎样的中英同义词、常见问法或用户表达找到。

`aliases` 是人工可审阅的数据，不由工具猜测后直接写入。旧卡不带此字段仍然合法，解析、格式化、索引和快照必须保留它。v0.7 将为当前库中与本机工作流相关的卡片补充少量已审阅别名，并把这些自然表达加入检索评测。

## v0.7.0：可解释的别名检索

### 行为

`rag_search` 与 CLI `search` 保持现有参数和结果字段，同时使用下列顺序排序：

1. 完整查询短语命中标题、别名、标签、来源或正文。
2. 查询中的有效词/中文双字词被多少个不同词命中；覆盖越完整，优先级越高。
3. 命中的字段质量：标题最高，别名和标签其次，来源再次，正文最低。
4. 现有的筛选条件、复核新鲜度、置信度和稳定的 ID 排序。

常见中文功能词和孤立的一字符噪声不计入覆盖分，避免“的”“一半”等偶然重叠压过真正相关的卡片。没有任一有效词或完整短语重叠的卡仍不会返回。

搜索结果的 `match_reason` 改为可读的证据列表，例如 `alias phrase; title tokens 2/3`。`score` 仍保留为内部可比较的数值；不承诺其跨版本绝对稳定。

### 验收

- 旧卡、现有九个工具、JSON 和紧凑文本输出仍兼容。
- 自然表达回归集中，“网络共享路径来源检查”“知识库写到一半中断怎么处理”“备份没有带上新工具”分别在 Top-1 找到正确卡。
- 无词面或别名重叠的查询仍返回空结果。
- 别名解析、校验、索引往返、CJK 覆盖排序与格式化均有临时根测试。

## v0.8.0：任务开工简报

新增只读 MCP 工具 `rag_brief` 和 CLI `brief`。输入 `query`，可选 `project`、`limit` 与 `include_deprecated`；它复用 v0.7 搜索，不创建知识根、不更新任何卡片或索引。

输出包含：

- 与任务最相关的卡片摘要、ID、命中理由和来源；
- 从卡片状态、复核日期和当前 `rag_health` 派生的复核提醒；
- 明确的边界提示：命令、路径、端口、版本和外部事实仍须在使用前复验；
- 可供模型直接阅读的紧凑文本模式，以及脚本使用的 JSON 模式。

如果库有读取完整性错误或待恢复事务，简报如实返回错误/警告，不假装给出可靠建议。它只组织已有证据，不能生成新知识或替代人工判断。

### 验收

- 简报对真实库与临时根都不产生写入。
- 简报可将自然表达的 Top 结果与复核提醒一起返回。
- MCP 参数错误在不初始化空知识根的情况下受控报错。
- 现有九工具 smoke 继续通过，新增十工具 smoke 覆盖 `rag_brief`。

## v0.9.0：维护计划

新增只读 MCP 工具 `rag_maintenance_plan` 和 CLI `maintenance-plan`。它扫描本机知识根并产出按优先级排序的建议，不执行建议。

建议类型和证据固定为：

- `verify`: 距上次复核超过阈值的卡；
- `source_missing`: 本地绝对来源不存在；
- `add_aliases`: 缺少别名、且处于 active/needs_verification 状态的卡；
- `possible_duplicate`: 规范化标题或别名完全相同的卡。只有明确证据才报告，不能用模糊相似度指控重复。

每项建议必须包含卡片 ID、优先级、理由和安全的下一步。维护计划不会调用 `rag_mark_verified`、`rag_upsert`、`rag_finish_task`、`rag_reindex` 或删除文件；由用户或 Codex 在复核证据后决定是否执行。

### 验收

- 临时根覆盖每一种建议、空计划、错误卡/待恢复事务与绝对来源边界。
- 重复判断不因普通词重叠产生误报。
- 真实库的计划可读、无写入；已健康的字段不被错误标为问题。
- 所有 CLI/MCP 参数均经 schema 校验，文档和便携工作区同步更新。

## 模块边界

- `cardStore.mjs`：唯一负责卡片别名、排序、简报和维护计划的纯数据逻辑；所有新读取接口先经过现有事务和完整性边界。
- `cli.mjs`：只解析并展示 CLI 命令，不重复业务规则。
- `server.mjs`：声明 schema、转发工具调用并维持 JSON-RPC 隔离。
- `test/`：临时根回归、MCP 协议回归与真实库检索评测；真实知识卡的内容变更另行受审阅。

## 发布与验证顺序

每个小版本独立完成：源码/测试先行，随后更新 README、SPEC、MCP 配置允许列表和 Portable Workspace，最后运行 Node 测试、MCP smoke、真实库 `validate`/`reindex`/`health`、检索评测及便携副本验证。每次对真实库写入前先生成不覆盖的快照。

v0.7 先发布，因为 v0.8 和 v0.9 都依赖稳定、可解释的搜索结果；v0.8 和 v0.9 只读，因此不会绕过 v0.6 的事务写入边界。
